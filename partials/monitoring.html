A Monitoring application with Flask, psutil, Docker, ECR, and EKS Orchestration

  1. View GitHub

Introdcution

The Cloud-Native Web Voting Application is a modern, scalable, and secure solution that enables users to cast votes for their preferred programming language through a web interface. Leveraging Kubernetes for orchestration and management, the application ensures high availability, scalability, and efficient resource utilization.

Technical Stack:

  1. Part 1: Deploying the Flask application locally
    1. Step 1: Clone the code: Clone the code from the repository
    2. Step 2: Install dependencies: The application uses the psutil and Flask, Plotly, boto3 libraries. Install them using pip
    3. Step 3: Run the applicationTo run the application, navigate to the root directory of the project and execute the following command: python3 app.py
  2. Part 2: Dockerizing the Flask application
    1. Step 1: Create a Dockerfile: Create a Dockerfile in the root directory for app.py
    2. Step 2: Build the Docker image:To build the Docker image, execute the following command: docker run -p 5000:5000
    3. Step 3: Run the Docker container
  3. Part 3: Pushing the Docker image to ECR
    1. Step 1: Create an ECR repository: Create an ECR repository using ecr.py
    2. Step 2: Push the Docker image to ECR : Push the Docker image to ECR using the push commands on the console: docker push :
  4. Part 4: Creating an EKS cluster and deploying the app using Python
    1. Step 1: Create an EKS cluster: Create an EKS cluster and add node group
    2. Step 2: Create a node group: Create a node group in the EKS cluster.
    3. Step 3: Create deployment and service Once you run this file by running “python3 eks.py” deployment and service will be created. Check by running following commands: kubectl get deployment -n default (check deployments)
      kubectl get service -n default (check service)
      kubectl get pods -n default (to check the pods)
